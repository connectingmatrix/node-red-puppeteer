Description

Navigate to a URL, take a screenshot or generate a PDF from the page

Inputs

Page ID string
The pageId of the page on which the action is to be performed
actionType string
The type of action to be performed - "navigate", "generatePDF" or "screenshot", where each corresponds to their obvious meanings

If actionType is "navigate"

-
  • URL - The URL to navigate to
  • waitUntil - how long the node should wait before forwarding the message to the next node. What these options mean -
    • load - consider navigation to be finished when the load event is fired.
    • domcontentloaded - consider navigation to be finished when the DOMContentLoaded event is fired.
    • networkidle0 - consider navigation to be finished when there are no more than 0 network connections for at least 500 ms.
    • networkidle2 - consider navigation to be finished when there are no more than 2 network connections for at least 500 ms.
  • viewport - if "default", the new page will open in 1600x900 dimensions. If "custom", you can specify your own dimensions

If actionType is "generatePDF"

-

This action only works if the browser is opened in headless mode.

  • Ouput file path - the file path that the PDF should be saved to
  • Scale - the scale of the page at which the PDF should be generated
  • Header HTML - custom HTML header (if any) that the PDF should be given
  • Footer HTML - custom HTML footer (if any) that the PDF should be given
  • Orientation - landscape or portrait. Obvious meaning
  • Page format - the page dimensions of the PDF
  • useCustomMargins - if custom margins should be used. If yes, you can specify your margins along with units, like "1cm"

If actionType is "screenshot"

-

This action only works if the browser is opened in headless mode.

  • Ouput file path - the file path that the PDF should be saved to
  • Type - the output image format (png, jpeg or webp)
  • Quality - image quality on a scale of 1 to 100, 100 being the original. Not supported if type is png.
  • captureArea - what part of the page should be captured What these options mean -
    • visiblePage - the part of the page that's visible at the moment
    • fullScrollablePage - the entire scrollable page (i.e., including the part that is not visible)
    • custom - specify the coordinates of the top-left corner of the screenshot area, along with its height and width

Outputs

This node does not have any outputs. If the page operation is successful, it simply forwards the message.
